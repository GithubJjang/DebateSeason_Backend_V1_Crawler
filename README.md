# 토론철 웹 크롤러
: 뉴스, 커뮤니티 정보 크롤링.</br>
이를 가공하여 향후 사용자들에게 다양한 서비스 제공(뉴스, 커뮤니티, AI 추천 등)

## 1. 크롤러 설계 및 개발
- 1-1. 운영 비용 절감을 위한 설계 </br>
: 24시간 가동을 기준으로, AWS t3.medium 인스턴스로 크롤러를 가동할 경우 한달 60,000원 지출. (이는 전체 예산의 60% 차지) </br>
비용 절감이 최우선 목표. </br>
GCP e2(1 공유 코어, 4GB)를 선정하여 싱글 스레드, 순차 처리, 스로틀링을 적용하여 크롤링 작업 안정성 확보. </br>
작업 시간이 증가했지만, 비용 23% 절감으로 최우선 목표 달성.

- 1-2. 크롤러 장애 복구 자동화 </br>
: 장애로 인한 서버 재부팅 시, 중단 지점부터 복구하기 위한 선제적 설계 필요성 인식. </br>
따라서 각 크롤링 작업 완료 시 상태 정보를 파일로 저장( 이미 존재할 경우, lastModified만 업데이트 )하며, 재부팅 시 최신 lastModified를 기준으로 마지막 체크포인트부터 작업 자동 재개. </br>
그 결과, 장애 발생시 중단 지점부터 성공적으로 작업 재개. </br>
